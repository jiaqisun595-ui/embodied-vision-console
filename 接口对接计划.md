## 1. 核心设计原则：一套组件，两种数据源

这三块预留接口都遵循同一个模式：

| 模块         | Mock 源                              | 真实源                             | 切换开关                          |
| ------------ | ------------------------------------ | ---------------------------------- | --------------------------------- |
| 思考流       | `MOCK_THOUGHTS` 数组循环             | `GET /api/brain/latest` 轮询       | `ENDPOINTS.thoughtStream` 是否为空 |
| RGB / Depth  | 空 → 显示 "NO SIGNAL"                | `GET /rgb` · `/depth`（MJPEG）     | `ENDPOINTS.rgb` / `.depth` 是否为空 |
| 3D 世界模型  | `public/mock/world.glb` 本地文件     | `GET /api/world/latest` 轮询       | `ENDPOINTS.worldModel` 是否为空    |

**关键理念**：`config.ts` 里的 URL 为空字符串时跑 mock，填了 URL 就切换到真实后端。组件代码本身对"现在是 mock 还是真实"无感知——这叫 **配置驱动 / feature flag 模式**，是前端对接外部接口最稳的打法。

## 2. RGB / Depth 视频流

### 2.1 老师侧提供的契约

```
GET http://<IP>:<PORT>/rgb     · 实时 RGB 彩色视频流
GET http://<IP>:<PORT>/depth   · 实时深度图（Jet 色彩映射，越红越近）
```

- 返回类型：`multipart/x-mixed-replace`（MJPEG）
- 分辨率：640×480
- 测试地址：`10.100.129.68:8001`

### 2.2 概念扫盲（下个对话里可以不复述，这里留给你回顾）

- **MJPEG ≠ 视频流的另一种东西**，它就是"视频流"的一种实现。服务器通过一条长 HTTP 连接不停推 JPEG 图片，浏览器把响应头识别为 `multipart/x-mixed-replace` 后就进入"每收到一张新图就替换显示"的模式。
- **关键奇迹**：MJPEG 可以直接塞进 `<img src="http://ip/rgb">`，浏览器原生支持，前端零代码。
- `/rgb` 和 `/depth` 的传输、协议、响应头、帧格式**完全相同**，唯一区别是像素内容（真实彩色 vs 染色深度）。因此本地 mock 时不需要为它们分别写两套逻辑。

### 2.3 当前状态

- `src/config.ts`：`ENDPOINTS.rgb` 和 `ENDPOINTS.depth` 已经填了 `http://10.100.129.68:8001/rgb` 和 `/depth`
- `RGBVideo.tsx` / `DepthVideo.tsx`：极简，就是把 URL 传给共用的 `StreamFrame`
- `StreamFrame.tsx`：三状态（CONNECTING → LIVE → OFFLINE），失败后按 `RETRY_INTERVAL_MS` 自动重试；离线时显示 "NO SIGNAL / waiting for <label>"，不再依赖任何静态占位图

### 2.4 待办 · 本地 mock 方案

目标：在老师机器没开或者本地开发时，左右两个框也要有东西在播。两条路线二选一：

#### 路线 A · 本地静态动图（最省事，推荐先走）

- **做什么**：把两段示例素材转成 `.gif`（或 animated `.webp`），放到 `public/mock/rgb.gif` 和 `public/mock/depth.gif`
- **改代码**：无——直接把 `config.ts` 里的 `ENDPOINTS.rgb` 改成 `"/embodied-vision-console/mock/rgb.gif"`（注意要带 Vite 的 base 前缀），`depth` 同理
- **切换回真实接口**：只改 `config.ts` 那两行 URL
- **优点**：零依赖、关网能用、前端代码一个字都不动
- **缺点**：不是真正的 MJPEG，不能提前踩到网络 / CORS / 断流的坑

#### 路线 B · 本地跑一个假 MJPEG 服务（最贴近真实）

- **做什么**：写一个 `scripts/mock_mjpeg_server.py`，用 Flask + OpenCV，大概 30 行代码：
  - 监听 `localhost:8001`
  - 读一段本地 `.mp4` 素材
  - 循环解帧 → `cv2.imencode('.jpg', frame)` → 按 multipart 包头往外推
  - `/depth` 路由在送出前先对每帧做一次 `cv2.applyColorMap(gray, cv2.COLORMAP_JET)`，伪造深度效果
- **改代码**：`config.ts` 里把两个 URL 改成 `http://localhost:8001/rgb` 和 `/depth`，其余不动
- **切换回真实接口**：把 `localhost:8001` 换成 `10.100.129.68:8001`
- **优点**：从响应头到 parse 行为全链路和真机一致，`StreamFrame` 的状态流转、重连逻辑都能真实测到
- **缺点**：需要装 Python + `opencv-python` + `flask`（三个 pip 命令）；展示时终端要一直开着

### 2.5 新对话里的执行步骤

1. 确认当前 `config.ts` 的 RGB/depth URL 状态
2. 和用户确认走 A 还是 B（或两者都要）
3. 如果路线 A：帮用户找或生成两段示例 gif；如果路线 B：写好 Python 服务 + 启动脚本
4. 更新 `config.ts` 指向 mock 源
5. `npm run dev` 验证两个画面都有
6. 顺便验证"清空 URL → 显示 NO SIGNAL"这个回退路径

## 3. 思考决策流（已完成初版，这里记录以备参考）

### 3.1 契约（老师确认）

```
GET http://<brain-host>:<port>/api/brain/latest
Content-Type: application/json
```

轮询频率：每 2–3 秒一次。返回体：

```json
{
  "role":    "brain" | "world" | "act" | "done",
  "label":   "parse_cmd",
  "content": "Heard: walk to the farthest chair",
  "code":    "POST /nav_aim { target: 'c3' }"    // 可选
}
```

字段说明：
- `role`：四种角色之一，决定卡片颜色（brain 品红 / world 青 / act 琥珀 / done 翠绿）
- `label`：短 tag，例如 `parse_cmd`、`nav_to`，显示在卡片右上角
- `content`：一句话描述，中英文均可
- `code`：可选，代码 / JSON / 命令片段，作为 `<pre>` 代码块追加在卡片下方

### 3.2 当前实现

- **`src/hooks/useBrainLatest.ts`**：`fetch` 轮询 hook，按 `THOUGHT_POLL_INTERVAL_MS`（默认 2500ms）调用一次；有类型守卫校验字段；用 `role+label+content+code` 做指纹去重，避免重复插卡；错误静默（避免展览崩溃）；端点为空时整个 hook 不启动
- **`src/components/ThoughtStream.tsx`**：两套数据源分支——`ENDPOINTS.thoughtStream` 为空时跑本地 `MOCK_THOUGHTS` 循环；非空时切到 `useBrainLatest`，新卡到达就追加到 `displayed`。右上角状态徽章：`MOCK` / `LIVE · WAITING`（端点已配但未连上）/ `LIVE`
- **`src/config.ts`**：`ENDPOINTS.thoughtStream` 目前为空字符串，契约注释写在对应字段上方

### 3.3 新对话里的执行步骤（如果需要切到真实）

1. 拿到老师的 brain host 和端口（例如 `http://10.100.129.68:9000`）
2. 在 `config.ts` 把 `thoughtStream` 改成那个 base URL（**不要**带 `/api/brain/latest` 后缀，hook 会自己拼）
3. 刷新页面，观察右上角徽章能从 `LIVE · WAITING` 跳到 `LIVE`
4. 如果撞 CORS，找老师在后端加 `Access-Control-Allow-Origin: *`

## 4. 3D 世界模型（右下右 · 这次的重头戏）

### 4.1 数据流跟前两者**完全不同**

这个坑必须先打清楚：

- **RGB/depth**：推流，服务器主动持续发，前端被动接。延迟毫秒级
- **思考流**：轮询，前端每 2–3 秒问一次"最新一句话是什么"。延迟秒级
- **3D 重建**：**请求—响应 + 文件**。机器人采一组图 → 老师的模型跑几秒到几十秒 → 吐出一个 `.glb` 文件。这是一个**离线计算任务**的产物，天然不可能像视频那样实时推流

所以不要用"视频流"的思路去想这个，要用"每隔一段时间有一个新文件产生，我把最新的那个取下来显示"的思路去想。

### 4.2 GLB 是什么

GLB 是 **glTF 格式的二进制打包版本**，一种 3D 模型文件格式，类比：

- `.jpg` / `.png` → 图片标准文件
- `.mp4` → 视频标准文件
- `.glb` → 3D 模型标准文件（一个文件里装了几何体、材质、贴图、动画）

glTF 有两种形态：`.gltf`（JSON + 外部贴图文件）和 `.glb`（全部打包成一个二进制包）。老师给 GLB 就是图省事。

### 4.3 老师侧的契约（**尚未敲定**，这是我的建议）

建议让老师按这种模式提供：

**推荐 · 模式 A（轮询最新 GLB URL）**

```
GET http://<host>:<port>/api/world/latest
Content-Type: application/json

→ {
    "url":       "http://.../meshes/20260409_153022.glb",
    "timestamp": 1744200622
  }
```

前端每 N 秒请求一次，发现 `timestamp` 变了就换模型。

或者更偷懒的版本——老师直接用这个 URL 返回 GLB 字节流本身（`Content-Type: model/gltf-binary`），前端检测 `Last-Modified` 头是否变了来决定要不要重绘。

**为什么推荐模式 A**：
1. 和思考流 `/api/brain/latest` 同构，代码模式可以复用，未来维护省心
2. 对老师那边要求最低——他只要把每次算完的文件扔到一个固定 URL 就行
3. 如果后面老师改成 WebSocket 推送，只要改 `useWorldLatest` hook 内部，`WorldModel3D` 组件不用动

对接老师的话术就一句：**"能不能给我一个 `GET /api/world/latest`，返回最新那次重建出来的 GLB 文件？可以直接返回二进制，或者返回一个 JSON 里包含下载 URL。"**

### 4.4 前端怎么在网页上显示 GLB

浏览器**原生不认识** `.glb`，需要一个 3D 渲染器解析它画到 `<canvas>` 上。可选工具：

#### 推荐 · `<model-viewer>`（Google 出的 Web Component）

```html
<model-viewer
  src="/embodied-vision-console/mock/world.glb"
  camera-controls
  auto-rotate
  auto-rotate-delay="0"
  rotation-per-second="15deg"
  exposure="1"
  shadow-intensity="0.6"
></model-viewer>
```

- 一行标签搞定，自带鼠标拖拽、自动旋转、环境光、HDR 反射
- 只要 `import '@google/model-viewer'` 一次，整个 React 项目都能用 `<model-viewer>` 标签
- TypeScript 里需要做一次全局类型声明，否则 JSX 会报 "unknown tag"（具体见 4.6）

#### 备选 · Three.js / react-three-fiber

能力最强，但要写几十行代码搭场景、相机、灯光、GLTFLoader、render loop。除非后面要做复杂交互（点击高亮某个物体），否则没必要。

**结论**：先用 `<model-viewer>` 完全够用。

### 4.5 本地 mock 比前两者还简单

因为 GLB 本质就是**一个普通文件**，mock 不需要任何假服务器：

1. 去 [KhronosGroup/glTF-Sample-Assets](https://github.com/KhronosGroup/glTF-Sample-Assets) 下一个示例模型
   - `DamagedHelmet.glb`：反光战损头盔，Khronos 官方门面，科幻 HUD 感很强（推荐）
   - `Sponza.glb`：建筑室内场景，更贴近机器人重建的结果
2. 丢到 `public/mock/world.glb`
3. 组件的 `src` 指过去即可

Vite dev server 会把 `public/` 下的文件按原样 serve，等效于"有一个后端接口返回了这个 GLB"。

#### Step 5 · 新写 `src/hooks/useWorldLatest.ts`

```ts
// 轮询 /api/world/latest 获取最新 GLB URL
// 为空时返回本地 mock 文件路径
import { useEffect, useRef, useState } from "react";
import { ENDPOINTS, WORLD_POLL_INTERVAL_MS } from "@/config";

const MOCK_GLB = `${import.meta.env.BASE_URL}mock/world.glb`;

export const useWorldLatest = () => {
  const [modelUrl, setModelUrl] = useState<string>(
    ENDPOINTS.worldModel ? "" : MOCK_GLB,
  );
  const [connected, setConnected] = useState(false);
  const lastTsRef = useRef<number>(0);

  useEffect(() => {
    if (!ENDPOINTS.worldModel) return;
    const endpoint = `${ENDPOINTS.worldModel.replace(/\/$/, "")}/api/world/latest`;

    const tick = async () => {
      try {
        const res = await fetch(endpoint, { cache: "no-store" });
        if (!res.ok) return;
        const json = await res.json();
        if (typeof json?.url !== "string") return;
        setConnected(true);
        const ts = typeof json.timestamp === "number" ? json.timestamp : Date.now();
        if (ts === lastTsRef.current) return;
        lastTsRef.current = ts;
        setModelUrl(json.url);
      } catch {
        // 静默，下一 tick 自愈
      }
    };
    tick();
    const id = window.setInterval(tick, WORLD_POLL_INTERVAL_MS);
    return () => window.clearInterval(id);
  }, []);

  return { modelUrl, connected };
};
```

## 5. 概念速查（新对话里可以让 Claude 直接跳过这节）

这是之前几轮对话里讨论过的、你说想补的概念清单，按优先级：

**高优先级**
- **HTTP 基础**：GET/POST/PUT/DELETE、状态码、请求头 / 响应头、`Content-Type` 的作用
- **什么是"接口"**：URL + 方法 + 请求参数 + 响应格式 四件套
- **同源策略 / CORS**：本地 dev server 访问远程接口可能撞的坑；`<img>` 比 `fetch` 限制宽松

**中优先级**
- **流式 vs 请求—响应**：MJPEG 用 `<img>` 直接连 vs GLB 用 fetch 拉
- **Content-Type / MIME 类型**：`.glb → model/gltf-binary`、`.mp4 → video/mp4`、`.json → application/json`
- **静态文件 vs 动态接口**：`public/mock/world.glb` 是静态文件；`/api/world/latest` 是动态接口
- **端口**：一台机器同时跑多个服务靠端口区分（Vite dev 默认 5173，mock MJPEG 服务可以占 8001）
- **轮询（polling）**：定时请求"最新状态"接口，我们思考流和 world model 都用这个模式

**低优先级（以后遇到再学）**
- 编码 / 压缩（H.264、JPEG 压缩率）
- WebSocket / Server-Sent Events（轮询的替代品）
- RTSP / HLS / WebRTC 各自适用场景
- Three.js / WebGL 3D 渲染基础
