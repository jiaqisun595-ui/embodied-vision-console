## 0. 一页纸背景（新对话里 Claude 先读这段）

### 项目是什么

下周具身智能大会展厅现场，四足机器人会执行观众的语音指令（例如"先走到最远的椅子，再走到出发前身后的桌子"）。机器人身后有一块 1920×1080 的大屏，**这个大屏要展示的就是我在做的这个网页**，让观众直观看到机器人"脑子里和眼睛里"正在发生什么——相信它在真正思考和感知，而不是在跑预录脚本。

### 我在这次任务中的角色

我只做**前端**这一个网页。不涉及任何后端、不涉及部署、不涉及模型训练。
我的活就是：**写一个 HTML/React 页面，留好三个视频槽位和一个思考流区域，老师们把接口 URL 给我之后我填到 config 里。**

### 项目仓库

GitHub：`https://github.com/jiaqisun595-ui/embodied-vision-console.git`
技术栈：React + TypeScript + Vite + Tailwind + shadcn/ui
当前状态：有一个旧版本，按"THINKING / PERCEPTION 两种模式互斥切换"的思路做的。**这个思路已经被老师们在会上否决了**，需要重构。

## 1. 最终形态（老师们在会议里拍板的版本）

**一块 1920×1080 横屏，单页面，不再有模式切换，四个区域同时运转。**

```
┌─────────────────────────────────────────────────────────────┐
│  TopBar：项目标题 + 当前指令文字（英文）                    │
├──────────────────┬──────────────────────────────────────────┤
│                  │                                          │
│                  │    ┌───────────────────────────────┐     │
│                  │    │                               │     │
│                  │    │   Slot 1: RGB 视频（凯哥）    │     │
│   左栏 · 决策流  │    │                               │     │
│   ThoughtStream  │    └───────────────────────────────┘     │
│   （一条一条     │                                          │
│    往下冒的      │    ┌──────────────┬─────────────────┐    │
│    对话卡片）    │    │  Slot 2      │   Slot 3        │    │
│                  │    │  深度图      │   世界模型3D    │    │
│                  │    │  （凯哥）    │   （lyb 老师）  │    │
│                  │    └──────────────┴─────────────────┘    │
│                  │                                          │
├──────────────────┴──────────────────────────────────────────┤
│  BottomBar（可选）：状态栏                                  │
└─────────────────────────────────────────────────────────────┘
```

**左栏 ≈ 1/3 宽，右栏 ≈ 2/3 宽；右栏内：上方 RGB 大块，下方深度图和世界模型 3D 并列。**
最终比例可以微调，但这是基本形态。

## 3. 分阶段行动路径

### 阶段 0 · 心里有底（已完成✅）

- 最终形态已确认：单屏 + 左思考流 + 右三视频槽位
- 验收标准：
  1. 本地 `npm run dev` 能打开完整布局
  2. 三个视频槽位在接口加载失败时**显示占位图/占位视频**，不会空白
  3. 左侧思维流在 mock 数据下持续滚动
  4. 全屏 F11 后比例不乱
  5. `src/config.ts` 里集中配置三个接口 URL（方便现场改）
  6. README 写清楚怎么跑、接口改哪里、现场注意事项

### 阶段 1 · 本地重构布局（当前优先任务）

**分支策略**

```bash
git checkout -b feature/single-screen-layout
```

**第 1 步 · 搭骨架**

在 `src/pages/Index.tsx` 里用 Tailwind grid/flex 把四个区域的外壳画出来。先用纯色块 + 文字标签占位（"RGB 区域""深度区域""世界模型区域""思维流区域"），不纠结样式。跑起来看比例和全屏效果。

**第 2 步 · 抠出思维流组件**

新建 `src/components/ThoughtStream.tsx`。把现有 `ThinkingMode.tsx` 里"卡片一条条滑入"那段逻辑搬过去，做成独立的常驻组件。不再受 mode 控制，一直跑。

**第 3 步 · 建三个视频组件 + 配置文件**

新建：
- `src/components/RGBVideo.tsx`
- `src/components/DepthVideo.tsx`
- `src/components/WorldModel3D.tsx`

每个组件内部就是一个 `<img>`，带 `onError` 兜底到占位图。

新建 `src/config.ts`：

```ts
export const ENDPOINTS = {
  rgb: "http://10.100.129.68:8001/rgb",
  depth: "http://10.100.129.68:8001/depth",
  worldModel3D: "", // 等 lyb 老师
  thoughtStream: "", // 等一群老师
};

export const FALLBACKS = {
  rgb: "/mock/rgb-placeholder.jpg",
  depth: "/mock/depth-placeholder.jpg",
  worldModel3D: "/mock/world-placeholder.jpg",
};
```

**这是整个项目的"总开关"。现场改 URL 只改这一个文件就行。**

**第 4 步 · 摘掉不用的代码（不要删，只是不在 Index 里引用）**

- `PerceptionMode.tsx` → 保留文件但从 Index 移除
- `SpatialCanvas.tsx` → 同上（老师们明确说短期不做实时点云地图）
- `timeline.ts` → 改成只驱动思维流卡片节奏，不再负责模式切换

**第 5 步 · 文案转英文**

所有界面能看到的中文换成英文（不做 i18n 切换，直接硬编码）。老师原话："纯英文界面就 OK，外国人多一些。"

### 阶段 2 · 打磨 mock（阶段 1 之后半天）

**目标：在没连机器人的情况下也让 demo 看起来很"活"。**

- 在 `public/mock/` 下准备占位素材：
  - 一段机器狗第一人称视角的 mp4/webm（RGB 占位）
  - 一段深度图风格视频（搜 "depth map video"）或静态图
  - 一段抽象 3D 点云旋转视频（世界模型占位）或静态图
- 编一版丰富的思维流 mock 脚本（约 60 秒循环），对标 PRD 里的 Cycle 1/2/3 节奏
- 微调样式：霓虹青 `#00E5FF`、深色底 `#0A0E1A`、等宽字体、微弱发光边框
- 全屏 F11 下检查一遍

**做完这一步就可以**录屏发群 + 同步发三条接口请求消息**。

### 阶段 3 · README 交接文档（阶段 2 之后 1 小时）

在仓库根目录写一份 `README.md`，必须包含：

1. **这是什么**（两句话）
2. **怎么跑起来**（`git clone` → `npm install` → `npm run dev` → F11）
3. **接口在哪改**（直接点到 `src/config.ts`，说明每个字段对应哪个区域、空着时会显示什么占位）
4. **展厅当天注意事项**：
   - 必须连机器人所在局域网
   - 用 Chrome
   - F11 全屏
   - 视频不显示时先看浏览器控制台的 CORS / mixed-content 报错
   - 如果老师在现场改 IP，只改 `src/config.ts` 一个文件

### 阶段 4 · 推 GitHub + 发消息

- `git push origin feature/single-screen-layout`
- 合并到 main（或让老师从分支拉也行）
- 群里发三条消息（见第 4 节）+ 录屏 demo
- 等待老师回复接口 URL，拿到后填进 `src/config.ts`，再推一次

**明确不用的接口**（不要被接口文档里一长串吓到）：
- SLAM 导航（`/start_relocation` / `/nav_aim` / `/pause` / `/resume`）→ 大脑调，不是前端
- `/tts` / `/mic` → 语音服务，不是前端
- 世界模型的"适配度分数"/"环境属性" → 研究中的评估器，短期跟网页无关

## 6. 关键原则（提醒自己）

1. **先做 mock 版本，不等任何人的接口**。老师们给接口是锦上添花，拿不到也能上。

3. **不做模式切换、不做 SLAM 点云地图**。这两个想法已经被老师们否掉了。
4. **界面纯英文**。
5. **遇到卡点就在群里 @ 对应的人**，不要自己憋。
